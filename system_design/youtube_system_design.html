Video Playback & Adaptive Streaming
Adaptive Bitrate Logic (ABR)
1

Initial Quality Estimation

Client measures RTT, available bandwidth, buffer level. Selects starting quality (typically 360p). Device type detection: mobile → cap 1080p, TV → allow 4K.

2

DASH / HLS Manifest Fetch

Player fetches MPD (DASH) or m3u8 (HLS) manifest listing all quality levels and segment URLs. Signed URLs with ~30min expiry for DRM-protected content.

MPEG-DASHHLS (Apple)CMAF
3

Segment Download Loop

Downloads 2–4s video segments. Keeps 30s buffer ahead. Switches quality on segment boundary when BW changes ≥15%.

4

CDN Edge Selection

Anycast DNS + GeoDNS routes to nearest PoP. If miss → origin shield → GCS. Cache TTL ~24h per segment. Edge cache hit rate: ~97%+.

5

DRM & License

Widevine (Android/Chrome), FairPlay (Apple), PlayReady (Windows/Xbox). License token issued at stream start, content key decrypted in secure enclave.

Widevine L1FairPlayPlayReady
6

Playback Telemetry

Client reports: rebuffer events, quality switches, TTFF, dropped frames, geo, ISP. Feeds into QoE dashboard and ABR model training.

Recommendation Engine
1

User State Collection

Watch history (last 30 days), search history, liked/disliked, subscriptions, explicit not-interested signals, co-views, dwell time per session.

2

Candidate Generation (Two-Tower)

User tower + video tower → dot product → approximate nearest-neighbor. Generates ~100–500 candidates from billions. GPU accelerated.

Two-Tower DNNScaNN ANN
3

Multi-Stage Ranking

Stage 1: Wide & Deep model on ~500 candidates → ~50. Stage 2: Deep neural ranking with 600+ features → final 20. Objectives: satisfaction, watch time, CTR.

4

Diversity & Freshness Injection

Diversity constraint: no 2+ videos same channel consecutively. Fresh content (< 24h) boosted. "New to you" exploration bucket.

5

Policy / Safety Post-filter

Remove borderline content, apply COPPA safe mode for kids, country-specific content restrictions, advertiser-friendly checks.

6

Response Cache & Serve

Pre-computed feed cached 60–120s per user. Incremental update on new activity. Real-time override for breaking news / trending.

Key Trade-offs & Design Decisions
DecisionChosen ApproachAlternativeReason
Video StorageObject store (GCS)HDFS, distributed FSInfinite scale, geo-replication, cost-efficient cold storage, managed service
Counter AccuracyApproximate (HyperLogLog)Exact countsExact view counts at scale require distributed transactions; ~2% error acceptable for UI
Metadata DBSpanner (globally consistent)Eventual-consistent NoSQLCreator operations (publish, visibility) need strong consistency; Spanner provides this globally
Encoding codecAV1 for new 4K+H.265 (HEVC)AV1 royalty-free, 50% better compression vs H.264, Google-backed. H.265 licensing costs at scale.
Recommendation latencyPre-computed feed + real-time rerankFully real-time generationFull real-time at 800M DAU infeasible; pre-compute saves 10×, rerank adds freshness/context
Search rankingHybrid BM25 + dense embeddingsBM25 only / dense onlyBM25 handles exact match / rare terms; dense handles semantic; hybrid beats either alone
View count timingBuffered → batch writeWrite-through per viewAt 800M DAU × 10 views = 8B writes/day → must batch. Accept ~minutes delay in count display.
CDN strategyGoogle CDN + ISP cachesThird-party CDN onlyOwning infra reduces cost at YouTube scale. ISP on-premise caches cut peering cost dramatically.
Live latency modeUser-selectable (normal/low/ultra)One fixed latencyNormal (30s) maximizes quality for most; ultra (<2s) for real-time interaction (gaming). CPU/CDN cost differs significantly.
Notification fan-outAsync Pub/Sub with batchingSynchronous at upload timeA channel with 100M subs sending 100M push notifications synchronously is infeasible. Async queue handles burst.
Dislike countHidden from public (2021)Show exact countDislike campaigns harmed creator mental health; ratio still fed to recommendation model internally
Comments sortTop-comments-first (ML ranked)ChronologicalMost relevant comments surface faster; improves engagement metrics; creator-configurable
Auto-captionsASR at processing timeOn-demand / no captionsPre-generating captions improves accessibility, boosts search indexing, and reduces real-time compute load
Shorts vs long-form feedSeparate ranking modelsUnified modelUser intent and engagement signals differ; shorts model optimizes swipe-through; long-form optimizes session depth
